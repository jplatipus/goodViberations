UI Specification

Introduction

The UI, described in this document, specifies the app activity classes which currently have no functionality specified. As development of this project progresses, UI class specifications are moved out of this document, into a functionality document which specifies the UI and the functionality of the activity.

The functionality specification documents are kept in the functionality folder. The file Functionality.html gives a detailed overview of the functionality specification documents.

The data structures used in this app are described in DataLayer.html

Activity design

  • All the activity classes should be in the package com.j3t.dataentryapp.ui.activities
  • All the activity classes should be in the folder src/main/java/com/j3t/dataentryapp/ui/activities
  • The title is displayed at the top of the activity's view.
  • The activity widgets are placed within a vertically scrolling view. This allows the activity's contents to be viewable by the user in portrait and landscape modes.
  • Each widget is a class member variable, this allows for functionality to be added to alter the widget's appearance at runtime.

Misc. definitions

Some terms used in this project are defined below:

Entry

An entry is defined to be a set of text fields, each field has a name and a value. Some fields are unique to an entry type, some are common to all entries. Some field values reflect a data type, and so have a syntax associated with them. For example:

  • email must have a valid email address syntax,
  • url must have a valid http url syntax.
  • name starts with an uppercase.
  • name is unique, name is case insensitive for purposes of uniqueness. For example name Name and NAME are all treated as the same in terms of sorting and uniqueness, so only one of name, Name or NAME can be present in the list of entries.
